# Функциональные требования: MVP KkalScan (v0.1)

Документ описывает, **что должен уметь** минимальный продукт KkalScan для market test в RuStore.

**Цель MVP:** проверить, готов ли пользователь **смотреть рекламу или платить 199 ₽/мес** после исчерпания бесплатных сканов.

## 1. Контекст

### 1.1. Позиционирование

> Сфоткал еду — получил калории и БЖУ за несколько секунд. **3 бесплатных скана каждый день.** Без регистрации. Без ИИ-диетолога и лишних экранов.

### 1.2. Главная гипотеза

Пользователь, который увидел ценность фото-скана, **либо досмотрит рекламу за +2 скана, либо купит Pro**, а не уйдёт навсегда.

### 1.3. Архитектура (логическая)

```
Android (RuStore)          Backend (91.207.75.72)     Внешние сервисы
─────────────────          ──────────────────────     ───────────────
Камера / галерея    →      POST /scan          →      Vision API (Gemini / GPT-4o-mini)
Экран результата    ←      JSON блюда+КБЖУ
Дневник «Сегодня»   ↔      GET/POST/DELETE /diary     SQLite/Postgres в Docker
Экран лимита        →      rate limit по device_id (на сервере)
Оплата Pro          →      http://91.207.75.72:8080/pay (Точка API, после HTTPS)
Реклама             →      Yandex Rewarded Ads
События             →      AppMetrica
```

**Дневник, лимиты сканов и учёт бонусов за рекламу — только на сервере.**  
Клиент хранит только `device_id` (UUID) и кэш экрана «Сегодня» для быстрого отображения (invalidate после мутаций).

### 1.4. Инфраструктура backend

| Параметр | Значение |
|----------|----------|
| Хост | `91.207.75.72` (`butov6101.hlab.kz`) |
| ОС | Ubuntu, Docker, nginx |
| Деплой | Docker-контейнер `kkalscan-api`, порт **8080** |
| URL API | `http://91.207.75.72:8080/api/v1/` |
| TLS | **Нет** на этапе MVP (HTTP по IP); DNS + HTTPS позже |
| БД | SQLite в volume **или** PostgreSQL в Docker (на выбор при реализации; для MVP достаточно SQLite) |

Базовый префикс API: `/api/v1/`

## 3. Экраны (6 штук)

| # | Экран | Назначение |
|---|-------|------------|
| E1 | **Скан** (камера / галерея) | Выбор фото еды — вызывается **FAB «+»**, не вкладка |
| E2 | **Результат** | Показать распознанные блюда, ккал, БЖУ; добавить в дневник |
| E3 | **Дневник «Сегодня»** | Список приёмов за день, сумма ккал |
| E4 | **Лимит / Paywall** | Предложить рекламу (+2) или подписку Pro |
| E5 | **Привязка аккаунта** | После оплаты Pro: VK / Yandex (можно «Позже») |
| E6 | **Профиль** | Статус (гость / Pro), остаток сканов, CTA Pro, подсказка про FAB «+» |

Нижняя навигация: **E3 Дневник** | **FAB «+»** (скан) | **E6 Профиль**.  
Стартовый экран — **E3**. FAB «+» → выбор фото → **E2** → возврат в E3.  
При `limit_hit` → **E4**. После активации Pro → **E5** (если аккаунт ещё не привязан). Подробнее: [auth-after-pro.md](./auth-after-pro.md).

## 5. Бизнес-правила

### 5.1. Счётчик сканов

```
free_scans_per_day     = 3
bonus_from_ad          = +2  (макс. 1 раз в день)
reset_at               = 00:00 локальное время устройства
pro                    = безлимит
```

Успешный скан = запрос к `/scan` вернул 200 и пользователь нажал «Добавить в дневник».  
Неуспешный скан (ошибка API) **не списывает** лимит.

### 5.2. Монетизация

| Тариф | Цена | Сканы |
|-------|------|-------|
| Free | 0 ₽ | 3/день + опционально +2 за рекламу |
| Pro | 199 ₽/мес | Безлимит |

Тариф: `pro_monthly_199` (199 ₽/мес, оплата через Точка API)

### 5.3. Пример дня free-пользователя

```
08:00  скан 1 (завтрак)     осталось 2
13:00  скан 2 (обед)        осталось 1
19:00  скан 3 (ужин)        осталось 0
19:30  скан 4 → E4 лимит
       → реклама → +2        осталось 2
21:00  скан 4 OK            осталось 1
       скан 5 OK            осталось 0
       скан 6 → E4 (только Pro или ждать завтра)
```

## 8. Критерии приёмки MVP (готово к публикации)

MVP считается готовым, когда выполнено **всё** ниже:

- [ ] Установка из RuStore на 3 реальных Android-устройствах из РФ
- [ ] Полный путь: установка → E3 → скан → результат → дневник на сервере
- [ ] Запись дневника видна после переустановки приложения (тот же `device_id`)
- [ ] 3 скана в день работают; 4-й открывает E4
- [ ] Rewarded video даёт +2 скана после досмотра
- [ ] Покупка Pro 199 ₽ снимает лимит (тестовая транзакция)
- [ ] Восстановление Pro через VK после привязки (новое устройство)
- [ ] E5 после покупки; «Позже» не отключает Pro на текущем телефоне
- [ ] События AppMetrica видны в отчёте
- [ ] При достижении потолка API — понятная ошибка, приложение не падает
- [ ] Карточка RuStore: иконка, 2–3 скрина, описание, политика конфиденциальности
- [ ] Модерация RuStore пройдена
